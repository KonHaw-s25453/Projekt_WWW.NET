================================================================================
  GAME LIBRARY – OGÓLNY OPIS PROJEKTU
================================================================================

TEMAT
------
„Game Library" to webowa aplikacja ASP.NET Core MVC do zarządzania biblioteką
gier komputerowych. Projekt prezentuje trzy relacje wiele-do-wielu, w tym
relację self-referencing dla znajomych.

TECHNOLOGIE
-----------
- Język programowania : C# / .NET 8
- Framework webowy    : ASP.NET Core MVC
- ORM                 : Entity Framework Core 8
- Baza danych         : SQLite (plik gamelibrary.db)
- Widoki              : Razor (.cshtml) + Bootstrap

GŁÓWNE ENCJE
------------
1. Player      – gracz (Id, Nick, Email)
2. Game        – gra (Id, Title, ReleaseYear)
3. Tag         – tag/kategoria (Id, Name)
4. PlayerGame  – tabela pośrednia Player ↔ Game (PlayerId, GameId, HoursPlayed)
5. Friend      – tabela pośrednia Player ↔ Player (PlayerId, FriendId)
6. GameTags    – tabela pośrednia Game ↔ Tag tworzona przez konfigurację EF Core

RELACJE M:N
-----------
1. Player ↔ Game (przez PlayerGame)
2. Game ↔ Tag (przez tabelę GameTags konfigurowaną w AppDbContext)
3. Player ↔ Player (przez Friend, relacja self-referencing)

FUNKCJONALNOŚCI
---------------
- CRUD dla graczy (PlayerController)
- CRUD dla gier (GameController)
- CRUD dla tagów (TagController)
- Strona główna z listą graczy i gier (HomeController)
- ViewComponent FriendList renderujący listę znajomych gracza

DANE STARTOWE (SEED)
--------------------
Podczas migracji dodawane są przykładowe rekordy:
- Tagi: RPG, FPS, Indie, OpenWorld
- Gracze: Jan, Anna
- Gry: Wiedźmin 3, Minecraft, Cyberpunk 2077
- Powiązania PlayerGame, GameTags i Friend

URUCHOMIENIE
------------
1. Przejdź do katalogu GameLibrary
2. Wykonaj: dotnet build GameLibrary.sln --nologo
3. Uruchom: dotnet run --project GameLibrary.csproj

Przy starcie aplikacji wywoływane jest db.Database.Migrate(), więc baza danych
jest tworzona/aktualizowana automatycznie.
